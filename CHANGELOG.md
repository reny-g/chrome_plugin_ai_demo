# 更新日志

本项目的版本更新记录维护在此文件中。

## [未发布]

### 优化

- 简历优化提示词升级至 2.3.0：两版简历重命名为「进取版 / 稳妥版」，消除原「投递版 / 面试版」暗示的分阶段使用误导，明确二选一定位。
- 简历优化提示词 2.2.0：进取版面向过筛做轻量包装与 JD 对齐，稳妥版保守可举证；新增反空话规则、`packaged` 变更类型。

### 新增

- 当前简历区域新增「复制」与「下载」按钮，展开预览时高度提升至约 480px，「查看」在展开后显示为「收起」。
- 设置页新增「服务商」预设下拉：选择 DeepSeek、OpenAI、Kimi、智谱 GLM、通义千问、火山豆包等服务商后，自动填入对应的接口地址与默认模型，仍可手动修改。
- 预设数据集中在 `config/providers.json`，新增服务商只需修改该配置文件。
- 新增通用配置加载工具 `config-loader.js`（读取扩展自带 JSON）。
- 配置加载失败时自动降级为纯手动填写，不影响设置页基本功能。

## [1.2.0] - 2026-06-14

### 新增

- 新增历史记录功能，网页摘要和简历优化两种模式各自保留最近 5 条生成结果。
- 顶部新增独立“历史”标签页，集中展示两种模式的记录。
- 点击历史记录可恢复到对应模式查看，复用原有的复制和下载能力。
- 支持单条删除和一键清空全部历史。
- 历史记录仅保存在浏览器本地，超出 5 条自动淘汰最旧记录，失败结果不入历史。

## [1.1.0] - 2026-06-09

### 新增

- 新增简历优化模式，并保留原有网页摘要模式。
- 支持上传、替换、查看和清除一份本地 Markdown 简历。
- 根据当前网页内容提取 JD 信息并生成两份完整 Markdown 简历：
  - 进阶简历：尽可能匹配岗位需求，对缺少事实依据的内容使用待补充提示。
  - 稳妥简历：仅基于原简历事实优化结构、重点和表达。
- 新增 JD 分析、差异说明、能力缺口和补充建议。
- 支持复制优化后的简历，并下载两份简历及分析报告。

### 优化

- 简历优化生成期间增加动态等待状态、已用时间和不确定进度动画，降低长请求被误判为卡死的概率。
- 收紧风险提醒的成立条件，区分生成可靠性风险与候选人信息缺口，并要求先进行语义等价判断。
- 优化生成提示词，要求尽量保留原简历的结构、标题层级、语气和排版风格。
- 将简历优化提示词拆分为独立版本化模块，便于后续迭代和问题诊断。
- 分层强化进阶版与稳妥版的事实边界，并规范无证据内容的待补充表达。
- 增加网页和简历提示词注入防护、精简 JSON 输出示例及输出前事实检查。
- 增加非招聘页面、内容截断、证据不足和事实风险提醒。
- 增加 AI JSON 输出结构及 JD 字段校验。
- 防止上传、替换或清除简历后，旧生成请求覆盖当前结果。
